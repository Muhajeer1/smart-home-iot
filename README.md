# Smart Home IoT DBMS

Smart Home IoT Data Management System for a database course project. The app stores homes, rooms, devices, sensors, sensor readings, events, alerts, and analytics reports.

Real IoT hardware is not connected in this demo. Sensor readings are generated by the built-in software simulator.

## Features

- User registration and login
- Admin, user, and technician roles
- Home, room, and device management
- Automatic default sensors for each device type
- Single and bulk sensor reading simulation
- Alerts for abnormal readings and device states
- Events log
- Analytics reports using joins, grouping, aggregation, and date filtering

## Requirements

- Node.js
- npm
- PostgreSQL

This project is built for PostgreSQL. Other database systems such as MySQL, SQLite, or MongoDB will not work without changing the backend database driver and SQL queries.

## Install PostgreSQL On Windows

Download PostgreSQL for Windows from the official PostgreSQL website:

```text
https://www.postgresql.org/download/windows/
```

Use the interactive Windows installer. During installation:

- Keep the default port: `5432`
- Set a password for the `postgres` user
- Remember this password because it must be added to `.env`
- pgAdmin can be installed too, but it is optional

After installation, open Command Prompt or PowerShell and check that PostgreSQL works:

```bash
psql --version
```

If `psql` is not recognized, use pgAdmin to create the database, or add PostgreSQL's `bin` folder to the Windows PATH. The folder usually looks like this:

```text
C:\Program Files\PostgreSQL\<version>\bin
```

## Setup

Clone the repository:

```bash
git clone https://github.com/Muhajeer1/smart-home-iot.git
cd smart-home-iot
```

Install dependencies:

```bash
npm install
```

Create a local environment file:

```bash
cp .env.example .env
```

Open `.env` and set your PostgreSQL credentials:

```env
DB_HOST=localhost
DB_USER=postgres
DB_PASSWORD=your_postgres_password
DB_NAME=smart_home_iot
DB_PORT=5432
```

## Database Setup

Create the database:

```bash
createdb smart_home_iot
```

Create the tables and default roles:

```bash
psql -d smart_home_iot -f database.sql
```

The default roles created by `database.sql` are:

```text
admin
user
technician
```

## Run The App

Start the server:

```bash
npm start
```

Open the app:

```text
http://localhost:3000
```

For development with auto-restart:

```bash
npm run dev
```

## Demo Flow

1. Register a new user.
2. Create a home.
3. Create rooms inside the home.
4. Add devices to rooms.
5. Open a device to view its automatically created sensors.
6. Open a sensor to view readings.
7. Use admin or technician dashboard to generate readings.
8. Generate a bulk dataset, for example 5000 readings.
9. Open alerts to see detected problems.
10. Open analytics to view reports.

## Analytics Reports

- Number of Devices per Home
- Latest Sensor Reading per Sensor
- Sensors with Abnormal Readings
- Alert Summary by Type for the Last 30 Days
- Average Reading Value by Sensor Type
- Number of Rooms per Home

## Alerts

Alerts are created by application logic:

- `abnormal_reading`: a sensor reading is outside the normal range
- `device_offline`: a device state is changed to offline
- `maintenance_required`: a device state is changed to maintenance

Alerts are different from readings. Readings are raw sensor measurements. Alerts are actionable problems that should be reviewed.

## Notes For Teammates

- Do not commit `.env`.
- Do not commit `node_modules`.
- Use `database.sql` to create your own local database.
- Data is local to each developer's PostgreSQL database.
- Bulk readings can be generated from the admin or technician dashboard.
